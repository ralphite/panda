# Repository Instructions

- Only create a commit when the user explicitly says `commit`.
- When creating a commit, use a very dry and succinct message.
- Always create or update a mini spec file when a change is feature-related.

## Mini Spec

A mini spec file is a `feature_name.spec.md` file in the `./spec` folder.

- Keep it very dry.
- Include anything that defines the product feature.
- Exclude tech details that can be derived from the spec.
- Exclude anything a coding agent can fill in.
